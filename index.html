16-Week CCDC Workshop

Hands-on Docker and Kubernetes training for CCDC. These workshops cover everything from setting up containers to locking them down for security.

Workshop Roadmap

Docker

Weeks 1-8

Start, run, and secure containers

Kubernetes

Weeks 9-16

Deploy, manage, and lock down clusters

Two 1-hour sessions a week: short talk, quick demo, hands-on lab, and a fast debrief.
Every week wraps up with a plug-and-play business-inject template for CCDC.

Detailed Schedule

Wk Track Focus Block Lecture & Demo Highlights Guided Lab (hands-on) Inject-Template Homework *
1 Docker – Deploy Fundamentals – images, containers, volumes, basic networking Run nginx & MySQL with named volumes; explain layered FS. Launch two containers, connect via bridge net, verify data persistence. Secure Container SOP – step-by-step launch checklist (source repo, tag pinning, volume rules).
2 Docker – Deploy Migrating Legacy Apps – app analysis, Dockerfile for traditional stacks (LAMP) Analyze host-installed DVWA; write Dockerfile, use Compose for DB. Deconstruct a simple PHP app and containerize it. Migration Plan Template – service discovery, data migration steps, config mapping.
3 Docker – Deploy Image Building & Registry – Dockerfile best-practice, multi-stage, SBOM/cosign Build rootless Go micro-service (<40 MB); push to private registry & sign. Class pairs build & pull each other's images, verify signature. Image Approval Form – required labels, SBOM section, vulnerability scan link.
4 Docker – Manage Compose & Multi-Service Ops – docker-compose.yml, scaling, env files, secrets Deploy WordPress + MariaDB stack; scale web; rotate DB password with Compose. Simulate DB failure, replace container, confirm recovery. Service Runbook – ports, scaling commands, backup location, rollback notes.
5 Docker – Manage Swarm & Troubleshooting – managers/workers, overlay net, logs, metrics 3-VM Swarm; rolling update on API service; use docker service ps, logs, Prometheus. Kill a worker; observe reschedule; capture metrics. Trouble-Shoot Playbook – common Swarm errors, log paths, escalation ladder.
6 Docker – Manage Advanced Networking & Storage - overlay/macvlan nets, volume drivers, backups Create macvlan net for direct LAN access; use volume plugin to backup data. Backup, destroy, and restore a stateful service's volume. Data Protection Policy – backup schedule, retention, test-restore cadence.
7 Docker – Secure Image/Host Scanning – Trivy, Grype, Dockle, CIS-Docker-Benchmark Scan previous images & host; fail image with CRITICAL vuln, rebuild fixed. Given 3 images with different vulnerabilities, use Trivy/Grype to scan, generate a risk report, and fix the highest severity CVE by rebuilding one image. Vulnerability Response Matrix – severity, SLA, patch workflow.
8 Docker – Secure Runtime Defense – seccomp, AppArmor, rootless, Falco rules Apply custom seccomp profile; deploy Falco, trigger suspicious exec; alert Slack. Write a custom Falco rule to detect when a package manager (e.g., 'apt') is run in a container. Trigger the event and verify the Falco alert. Incident Report Template – timeline, IOC list, containment steps.
9 K8s – Deploy Kubernetes Core with KIND – control plane, Pods, Deployments, Services Create KIND cluster; deploy guestbook; understand manifest files. Deploy the guestbook app, then scale the frontend to 5 replicas and change its Service type to NodePort. K8s Deployment Checklist – namespace naming, labels/annotations, resource limits.
10 K8s – Deploy RKE2 Installation – HA install, kube-vip, embedded etcd, Rancher import Provision 3-node RKE2 on Proxmox; snapshot etcd. Fail a control plane VM, confirm quorum. Cluster Build Doc – IP plan, install script, first-boot hardening flags.
11 K8s – Deploy Migrating to Kubernetes - from Docker Compose, Kompose, Helm charts Take Week 2 app; convert Compose file to K8s manifests; deploy. Migrate WordPress stack using Helm chart vs. kompose. K8s Migration Strategy - stateful vs stateless, config management, ingress planning.
12 K8s – Manage Workload Ops – rolling/blue-green, probes, HPA, ConfigMap/Secret updates Patch deployment, watch rollout; generate load to scale via HPA; config reload. Inject bad image, practice rollback. Patch & Rollback SOP – approval chain, verification, rollback buttons.
13 K8s – Manage Advanced Networking – Ingress, MetalLB, NetworkPolicy, Service Mesh intro Expose service via Ingress-NGINX; install MetalLB; create deny-all network policy. Deploy a two-tier app. Expose the frontend via Ingress, then write a NetworkPolicy that only allows traffic from the Ingress controller to the frontend. Network Policy Rulebook - default-deny posture, standard allow-lists for DNS/monitoring.
14 K8s – Manage Advanced Storage – PVC, StorageClass, StatefulSets, Velero Backups Install Longhorn; attach PV to Postgres; create StatefulSet. Take a snapshot of the Postgres volume using Longhorn. Then, simulate data corruption, restore from the snapshot, and verify data integrity. K8s Disaster Recovery Plan – RPO/RTO targets, backup schedules, restore validation steps.
15 K8s – Secure Scanning & Policy – kube-bench, kube-hunter, Kyverno/Gatekeeper, SealedSecrets Run CIS benchmark; enforce Kyverno "signed-images" & PSS restricted; break rule. Write a Kyverno policy that requires all pods to have an 'owner' label. Test it by deploying a pod without the label (observe block), then with it (observe success). Security Baseline Report – benchmark scores, failing controls, remediation plan.
16 K8s – Secure Runtime & Supply-Chain Defense – Falco-K8s, Cilium/hubble, cosign verify, Flux GitOps Git→build→cosign sign→Flux deploy; Red-Team injects cryptominer image; detection. Team blue-vs-red capstone scoring. Executive After-Action – attack narrative, control effectiveness, next steps.

*Templates = ≤2 pages, ready to hand to CCDC judges on inject arrival.

Tooling Bundle

  • Scanning: Trivy, Grype, Dockle, kube-bench, kube-hunter
  • Runtime: Falco, seccomp profiles, AppArmor, Cilium/Hubble
  • Supply-chain: cosign, Kyverno, Gatekeeper, FluxCD
  • Observability: Prometheus, Grafana, Loki

We keep it simple: deploy → manage → secure. Learn what you need to win against Red Team and ace those injects.
